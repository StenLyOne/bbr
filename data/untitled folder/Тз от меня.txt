Общие технические принципы:
1. Объект media
Везде, где присутствует объект media, он содержит поле type, которое может быть либо "image", либо "video". Это означает, что в зависимости от выбранного значения редактор загружает либо изображение, либо видео. На фронте необходимо реализовать условный рендеринг соответствующего элемента: <img /> или <video />.

2. Символ переноса строки |
В строковых значениях символ " | " используется как маркер переноса строки. Это сделано для реализации кастомной анимации текста. На фронтенде необходимо разбивать строку по этому символу и рендерить построчно с нужной анимацией.

3. Секция Latest (главная страница)
Раздел должен быть реализован через Instagram-плагин. Предпочтительный подход — использовать массив ссылок на посты. Это даст максимальный контроль над отображаемым контентом. Альтернатива — автоматическая загрузка последних видео из профиля, но она сложнее в реализации и менее предсказуема.

4. Поле content (rich text)
У каждого ивента и элемента портфолио должно быть поле content, в котором хранится rich text в HTML-формате. Это полноценный текстовый блок, редактируемый через визуальный редактор в админке. Сейчас его в json нет.

Структура owned-events.json:
5. Двойное назначение файла
Файл owned-events.json используется для двух страниц:

/our-owned-events — отображает список ивентов

/our-owned-events/[slug] — отображает детальную страницу конкретного ивента

6. Поле more_events
Последний блок на странице конкретного ивента ([slug]) называется more_events. Это массив slug, указывающий, какие другие ивенты следует отобразить в блоке «More Events».

7. Объекты внутри events[]
Каждый объект внутри массива events содержит полный набор данных для отрисовки страницы /our-owned-events/[slug], включая:

заголовки и подзаголовки

описание

gallery, stats_block, media_block, sponsors, cta

rich content

любые другие метаданные, необходимые для генерации страницы



Для последних страниц:

1. content: []
Тип: Массив объектов
Описание:
Используется для динамического контента. Позволяет добавлять неограниченное количество объектов (блоков)

2. media: { video_src: "" }
Тип: Объект
Описание:
Медиа-блок, в котором всегда предполагается использование только видео. Поле video_src содержит ссылку на видеоролик (например, URL YouTube/Vimeo или путь к файлу). Точно так же работает с image_src и т д

3. latest: {}
Тип: Объект
Описание:
Раздел предназначен исключительно для редактирования заголовка. Дополнительные элементы или карточки не добавляются,

4. some_of_our_work: {}
Тип: Объект
Описание:
Раздел для отображения названий уже существующих кейсов (работ). Используется исключительно для вывода ссылок/заголовков без добавления новых работ — все работы берутся из глобального массива.

5. Страница "Cookies & Privacy Policy"
Тип: Статическая страница
Описание:
Содержит только заголовок и rich text-блок с форматированным контентом (текст, ссылки, заголовки).
